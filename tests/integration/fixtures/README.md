# Integration Test Fixtures

This directory contains sample environment variable configurations for integration testing.

## Fixture Files

- `us1_database.env` - Database configuration (User Story 1)
- `us2_api_keys.env` - API keys and secrets (User Story 2)
- `us3_feature_flags.env` - Feature flags (User Story 3)
- `us4_system_config.env` - System configuration (User Story 4)
- `us5_nested_config.env` - Nested/complex configuration (User Story 5)
- `special_chars.env` - Variables with special characters
- `unicode.env` - Unicode variable names and values
- `type_conversion.env` - Type conversion test data

## Usage

These fixtures are used by integration tests to ensure consistent test data and reusable test scenarios.
